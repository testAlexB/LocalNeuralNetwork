# Запрос на peer review: BPE-токенизатор для FishingLLM

**Контекст:** Реализовали BPE-токенизатор для проекта FishingLLM (decoder-only transformer ~150M). Лекция 2 (теория токенизации) уже прошла ревью и одобрена. Теперь нужно проверить код.

## Файлы в репозитории

https://github.com/testAlexB/LocalNeuralNetwork

- `tests/test_tokenizer.py` — 19 тестов
- `docs/lecture-tokenizer-implementation.md` — документация решения

## Вопросы для ревью

1. **Корректность реализации:** Правильно ли обёрнут ByteLevelBPETokenizer? Есть ли проблемы с encode/decode roundtrip для русского текста?
2. **Нормализация:** Корректно ли добавлен NFC normalizer после train()? Не сбрасывается ли он при load()?
3. **Special tokens:** Правильно ли выбраны special tokens? Достаточно ли их для ChatML-формата?
4. **Save/Load:** Надёжен ли механизм сериализации через vocab.json + merges.txt?
5. **Тесты:** 19 тестов — все проходят. Какие ещё кейсы не покрыты? Нужен ли тест на длинные тексты, на batch encoding с разной длиной, на производительность?
6. **API дизайн:** Удобен ли интерфейс (train/encode/decode/save/load)? Не избыточен ли?
7. **Совместимость с attention:** Будет ли этот токенизатор корректно работать с реализованным ранее MultiHeadAttention? Нужно ли добавить pad_token_id в конфиг?
8. **Антипаттерны:** Есть ли проблемы с архитектурой, обработкой ошибок, граничными случаями?
